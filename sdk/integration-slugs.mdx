## How to Use

When generating a `link_token`, specify the desired service by using the appropriate slug from the lists above.

For example, if you want to integrate with BambooHR for HRIS, use the slug `bamboohr` in your request.

## Example

Here's an example of a request payload to generate a `link_token` for integrating with **BambooHR** as an **HRIS**:

```json
{
  "end_user_data": {
    "org_name": "Kunal Organization", // unique entity name
    "org_logo": "https://xyss.com/images",
    "origin_id": "2kh3ub45ad" // unique entity ID
  },
  "category": "HRIS", // category
  "integration": "bamboohr" // integration slug
}
```

For **ATS** integration with **Greenhouse**, use the slug `green-house`:

```json
{
  "end_user_data": {
    "org_name": "Kunal Organization", // unique entity name
    "org_logo": "https://xyss.com/images",
    "origin_id": "2kh3ub45ad" // unique entity ID
  },
  "category": "ATS", // category
  "integration": "green-house" // integration slug
}
```

By using the correct slugs, you can seamlessly connect with the desired HRIS or ATS service through BindBee's Unified APIs.
